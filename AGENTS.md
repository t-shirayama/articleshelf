# ReadStack Agents

このドキュメントは、ReadStackプロジェクトで想定されるエージェントや役割を整理したものです。

## 目的

- 開発・設計の方向性を共有する
- 機能追加やタスク分担のためのエージェントを明確にする
- 今後の改善やレビューに活用する

## 想定エージェント

### 1. Product Agent

- 画面構成やユーザー体験を設計する
- 記事のストックと読了履歴のフローを整理
- 画像・スクリーンショット貼り付けから資産化までの体験を検討

### 2. Design Agent

- UIモックアップや画面レイアウトを作成
- デスクトップ・モバイル双方のビジュアル設計
- 記事一覧、詳細、タグ管理、追加モーダルの見せ方を定義

### 3. Frontend Agent

- Vue.js を使って画面を実装
- フィルタリング、ステータス管理、フォーム入力を実装
- レスポンシブUIと入力ワークフローを整備

### 4. Backend Agent

- Java / Spring Boot で API を設計・実装
- PostgreSQL に記事データを永続化
- 読了日、タグ、メモ、ステータス管理のデータモデルを設計

### 5. QA Agent

- 動作確認とテスト観点を整理
- 画像やスクリーンショットからの入力操作を検証
- UXの一貫性や性能、検索機能の品質を確認

## 開発方針

- エージェントごとに領域を分け、段階的に機能を実装
- まずは基本的な記事追加 / 一覧 / 詳細 / ステータス切り替えを完成させる
- 次にタグ検索や読了履歴の表示を強化する
- 最終的に「読んだ記事を資産化する」体験を磨き上げる

## 追加メモ

- 画像貼り付けだけでなく手動入力にも対応する
- 記事の読了管理とタグ検索を中心にしたナビゲーションを作る
- Vue.js と Spring Boot の分離により、フロントエンドとバックエンドを独立して育成する
- PostgreSQL を使った信頼性ある永続化を前提とする
