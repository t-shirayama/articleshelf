# Architecture Decision Records

ArticleShelf の主要な設計判断、代替案、トレードオフを残す場所です。
現在仕様の詳細は specs / deployment / testing を正本とし、ADR では「なぜその判断をしたか」を中心に記録します。

## ADR 一覧

- [001: Auth token strategy](001-auth-token-strategy.md)
- [002: OGP SSRF protection](002-ogp-ssrf-protection.md)
- [003: Free tier deployment](003-free-tier-deployment.md)

## 書き方

- 背景: どの制約や課題に対する判断か
- 決定: 採用した方式
- 代替案: 検討したが採用しなかった方式
- トレードオフ: 得たものと残る制約
- 今後: 見直す条件や追加課題
