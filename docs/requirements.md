# ReadStack Requirements

## 1. 概要

ReadStackは、読んだ技術記事を「資産」として蓄積・管理するストックアプリです。記事のURLやタイトル、タグ、メモ、読了日を保存し、振り返りや学習のために活用できるようにします。

## 2. ターゲットユーザー

- 技術記事を日常的に読むエンジニア
- 学んだ内容を蓄積しておきたい人
- 「あとで読む」すら消化できない人
- Qiita / Zenn / Medium などをよく利用する人

## 3. 主な課題

- ブックマークが散らかりやすい
- 読んだ記事を忘れてしまう
- 「あとで読む」が消化されない
- メモと記事が分離している
- 読了履歴の振り返りが面倒

## 4. 解決すること

- 記事をアプリ内で一元管理する
- URLからOGPを取得して記事情報を補完する
- 読了ステータスと読了日で整理する
- タグによるカテゴリ分けを可能にする
- 記事ごとにメモを残せるようにする
- 検索とフィルタで必要な記事をすばやく見つける

## 5. MVP要件

- 記事を追加できる（URL / タイトル / タグ / メモ / 読了日）
- URLからOGPを取得して記事タイトルや概要を自動補完
- 未読 / 読了を切り替えられる
- 読了日を記録できる
- タグを追加 / 選択できる
- 記事一覧を表示できる
- 検索とタグフィルタが使える
- 記事詳細で内容を確認・編集できる

## 6. 非機能要件

- 技術スタック: Vue.js + Java(Spring Boot) + PostgreSQL
- レスポンシブ対応
- REST API でフロントエンドとバックエンドを分離
- 将来的な拡張を見据えた設計
- データ永続化を前提とした信頼性

## 7. 追加検討項目

- 画像やスクリーンショットの保存と参照
- OGP自動取得
- AI要約やハイライト
- ユーザー認証 / マルチユーザー対応
- クラウド同期
