# UI仕様

## 共通

- レスポンシブデザイン
- 明確なステータス表示
- 情報はカード・テーブル形式で整理
- 一覧画面に詳細パネルを同時表示せず、記事選択時は詳細画面へ遷移

## 記事一覧画面

- 検索バー、並び替えセレクト、追加ボタンを画面上部に配置
- 未読 / 読了 / すべて / お気に入りのフィルタとタグフィルタは左サイドバーに配置
- ページタイトルは左サイドバーで選択中の項目に合わせて切り替える
- 記事カード/リストにタイトル、URL、タグ、ステータスを表示
- 記事カード/リストにおすすめ度を星で表示する
- 記事カードのOGP画像はサムネイル領域が表示範囲に近づいた時だけ取得し、IndexedDBに保存した画像Blobを優先して表示する。未保存または取得不可の場合はプレースホルダーを表示する
- クリックで記事詳細へ遷移
- 記事カード上でお気に入り追加 / 解除ができる
- 記事カード上で削除操作ができ、削除前に確認ダイアログを表示する
- 検索バーの近くに並び替えセレクトを表示する
- 追加ボタンを常時表示
- 左サイドバー下部に、画像付きの学習継続メッセージを表示する
- 学習継続メッセージは約100件のコメントと30種類程度のSVGイラストモチーフを持ち、画面遷移や記事追加などのアクション時にローテーションする
- 記事数が多い場合は、画面全体ではなく記事一覧領域内をスクロールする
- タグ数が多い場合は、画面全体ではなくタグ領域内をスクロールする

## 記事詳細画面

- 記事ヘッダーにタイトル、URL、タグ、ステータスを表示
- URLリンクは実際のリンク先を保持したまま、表示幅を超える場合のみ見た目上 `...` で省略する
- 通常は閲覧モードで表示し、編集ボタン押下時のみ編集モードへ切り替える
- 閲覧モードではメイン領域に概要、タグ、メモを表示し、右側にメタ情報（読了日、favorite、おすすめ度）と操作ボタンを表示する
- 編集モードではタイトル、URL、概要、タグ、メモ、ステータス、読了日、おすすめ度を編集できる
- 右側のステータス、読了日、おすすめ度、削除の領域は、閲覧モードと編集モードで同じサイズと位置を維持する
- 右側のステータスと読了日は、閲覧モードでは編集時と同じUIを非活性表示にする
- 右側のおすすめ度は、閲覧モードと編集モードで星の開始位置と間隔を揃え、閲覧モードでは暗めの星で非活性表示にする
- タグ編集は追加モーダルと同様に既存タグ選択と新規入力に対応する
- 閲覧モード中もお気に入りはヘッダーから切り替えできる
- 削除操作では確認ダイアログを表示し、確定後に削除する

## 追加モーダル

- URL入力欄
- モーダルを開いた直後はURL入力欄に初期フォーカスする
- URLは必須項目として扱い、未入力時は保存せずエラーメッセージを表示する
- タイトル入力欄は任意項目として表示し、未入力時はURLから自動設定されることが分かる補足を表示する
- 過去のタグから選択でき、新しいタグも直接入力できるタグ選択/追加フォーム
- おすすめ度入力を星 5 段階で表示する
- おすすめ度入力は、hover / focus中の星までをまとめて強調表示する
- ステータス選択は表示しない
- 「あとで読む」チェックボックスを視認できる形で表示し、デフォルトでチェック済みにする
- 「あとで読む」チェックボックスと読了日入力は同じ行に配置し、高さと幅が揃うように表示する
- 読了日入力/日付ピッカーは追加モーダルと詳細編集で共通の見た目にし、値を上下中央、カレンダーアイコンを右端に表示する
- メモ入力欄
- 保存 / キャンセル
- 保存に失敗した場合は追加モーダル内にエラーメッセージを表示し、入力内容を保持する
- 重複 URL の場合は、登録済みの記事詳細へ移動する導線をエラーメッセージ内に表示する
- 保存後は一覧画面に留まり、自動では詳細画面へ遷移しない
