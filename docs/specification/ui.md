# UI仕様

## 共通

- レスポンシブデザイン
- 明確なステータス表示
- 情報はカード・テーブル形式で整理
- 一覧画面に詳細パネルを同時表示せず、記事選択時は詳細画面へ遷移

## 記事一覧画面

- 検索バー、並び替えセレクト、フィルタボタン、追加ボタンを画面上部に配置
- 未読 / 読了 / すべて / お気に入りのフィルタは左サイドバーに配置
- タグ、おすすめ度、登録日範囲、読了日範囲のフィルタは上部のフィルタモーダルに集約する
- フィルタボタン付近に適用中条件の要約チップを表示する
- ページタイトルは左サイドバーや上部フィルタで選択中の項目に合わせて切り替える
- 記事カード/リストにタイトル、URL、タグ、ステータスを表示
- 記事カード/リストにおすすめ度を星で表示する
- 記事カード/リストのタグはおすすめ度の横に表示する
- 記事カード/リストに登録日と読了日をおすすめ度・タグと同じ行の右端に高さを揃えて表示する
- 記事カードのOGP画像はQiitaやZennなどの横長OGPを見やすい大きめのサムネイルとして表示し、サムネイル領域が表示範囲に近づいた時だけ取得する。IndexedDBに保存した画像Blobを優先して表示し、未保存または取得不可の場合はプレースホルダーを表示する
- クリックで記事詳細へ遷移
- 記事カード上でお気に入り追加 / 解除ができる
- お気に入り状態の記事カードでは、ハートアイコンを赤で表示する
- 記事カード上のお気に入り追加 / 解除は一覧全体をローディング表示に戻さず、カード状態を即時反映して裏で保存する
- 記事カード上で未読 / 読了を切り替えるボタンを表示する
- ステータス切り替え後は、スナックバーに結果と「元に戻す」操作を表示する
- 記事カード上で削除操作ができ、削除前に確認ダイアログを表示する
- 検索バーの近くに並び替えセレクトを表示する
- フィルタモーダルではタグ複数選択、おすすめ度複数選択、登録日範囲、読了日範囲をまとめて変更できる
- おすすめ度フィルタは未設定を含めて左から低い順に選べる
- 登録日と読了日は、それぞれ個別に条件をクリアできる
- 追加ボタンを常時表示
- 左サイドバー下部に、画像付きの学習継続メッセージを表示する
- 学習継続メッセージは約100件のコメントと30種類程度のSVGイラストモチーフを持ち、画面遷移・記事追加・サイドメニュー選択などのアクション時にローテーションする
- 記事数が多い場合は、画面全体ではなく記事一覧領域内をスクロールする

## 記事詳細画面

- 記事ヘッダーにタイトル、URL、タグ、ステータスを表示
- URLリンクは実際のリンク先を保持したまま、表示幅を超える場合のみ見た目上 `...` で省略する
- 通常は閲覧モードで表示し、編集ボタン押下時のみ編集モードへ切り替える
- 閲覧モードではメイン領域に概要、タグ、メモを表示し、右側にメタ情報（読了日、登録日時、更新日時、favorite、おすすめ度）と操作ボタンを表示する
- 編集モードではタイトル、URL、概要、タグ、メモ、ステータス、読了日、おすすめ度を編集できる
- 右側のステータス、読了日、登録日時、更新日時、おすすめ度、削除の領域は、閲覧モードと編集モードで同じサイズと位置を維持する
- 右側のステータス選択は一覧上部の並び順セレクトと同じ共通セレクトスタイルで表示する
- 右側の読了日、登録日時、更新日時は同じ入力風UIで表示し、登録日時と更新日時は閲覧・編集どちらでも時刻つきで非活性表示する
- 右側のおすすめ度は、閲覧モードと編集モードで星の開始位置と間隔を揃え、閲覧モードでは暗めの星で非活性表示にする
- タグ編集は追加モーダルと同様に、選択済みタグの削除、既存タグからの追加、新規タグ入力に対応する
- 閲覧モード中もお気に入りはヘッダーから切り替えできる
- 詳細ヘッダーのお気に入り状態のハートアイコンは赤で表示する
- 削除操作では確認ダイアログを表示し、確定後に削除する

## 追加モーダル

- URL入力欄
- モーダルを開いた直後はURL入力欄に初期フォーカスする
- URLは必須項目として扱い、未入力時は保存せずエラーメッセージを表示する
- タイトル入力欄は任意項目として表示し、未入力時はURLから自動設定されることが分かる補足を表示する
- 選択済みタグ、過去のタグからの追加、新しいタグの直接入力を分けて表示するタグ編集フォーム
- おすすめ度入力を星 5 段階で表示する
- おすすめ度入力は、hover / focus中の星までをまとめて強調表示する
- ステータス選択は表示しない
- 「あとで読む」チェックボックスを視認できる形で表示し、デフォルトでチェック済みにする
- 「あとで読む」チェックボックスと読了日入力は同じ行に配置し、高さと幅が揃うように表示する
- 読了日入力/日付ピッカーは追加モーダルと詳細編集で共通の見た目にし、値を上下中央、カレンダーアイコンを右端に表示する
- メモ入力欄
- 保存 / キャンセル
- 保存に失敗した場合は追加モーダル内にエラーメッセージを表示し、入力内容を保持する
- 重複 URL の場合は、登録済みの記事詳細へ移動する導線をエラーメッセージ内に表示する
- 保存後は一覧画面に留まり、自動では詳細画面へ遷移しない
