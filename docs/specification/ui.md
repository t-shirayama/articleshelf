# UI仕様

## 共通

- レスポンシブデザイン
- 明確なステータス表示
- 情報はカード・テーブル形式で整理
- 一覧画面に詳細パネルを同時表示せず、記事選択時は詳細画面へ遷移

## 記事一覧画面

- アプリのファビコンは左サイドバー左上のブランドマークに合わせる
- 検索バー、並び替えセレクト、フィルタボタン、追加ボタンを画面上部に配置
- 未読 / 既読 / すべて / お気に入りのフィルタは左サイドバーに配置
- 左サイドバーの一覧フィルタの選択表示は記事一覧画面でのみ有効にし、カレンダー画面ではカレンダーのみを選択状態にする
- タグ、おすすめ度、登録日範囲、既読日範囲のフィルタは上部のフィルタモーダルに集約する
- フィルタボタン付近に適用中条件の要約チップを表示する
- ページタイトルは左サイドバーや上部フィルタで選択中の項目に合わせて切り替える
- 記事カード/リストにタイトル、URL、タグ、ステータスを表示
- 記事カードのタイトルは、ドメインや概要より先に目に入るよう強めのサイズと太さで表示する
- 記事カードのタイトルは1行固定とし、長い場合は `...` で省略する
- 記事カードの概要は最大2行で省略し、タグや日付のメタ行と重ならない固定領域で表示する
- 記事カードの概要やメモが未入力の場合、概要欄は空欄で表示する
- 記事カード/リストにおすすめ度を星で表示する
- 記事カード/リストのタグはおすすめ度の横に表示する
- 記事カード/リストのタグは最大3件まで表示し、4件目以降は `+N` のチップにまとめる
- 記事カード/リストの未読 / 既読ステータスバッジは登録日と既読日のすぐ左に配置する
- 記事カード/リストに登録日と既読日をおすすめ度・タグ・ステータスと同じ行の右端に高さを揃えて表示する
- 記事カード/リストの登録日と既読日は上下中央を揃えて表示し、既読日が未設定の場合は「未設定」と表示する
- 記事カード/リストの登録日と既読日は長い方に合わせた同じ最小幅で表示する
- 記事カードのOGP画像はQiitaやZennなどの横長OGPを見やすい大きめのサムネイルとして表示し、サムネイル領域が表示範囲に近づいた時だけ取得する。IndexedDBに保存した画像Blobを優先して表示し、未保存または取得不可の場合はプレースホルダーを表示する
- クリックで記事詳細へ遷移
- 記事カード上でお気に入り追加 / 解除ができる
- お気に入り状態の記事カードでは、ハートアイコンを赤で表示する
- 記事カード上のお気に入り追加 / 解除は一覧全体をローディング表示に戻さず、カード状態を即時反映して裏で保存する
- 記事カード右上のアクション群に未読 / 既読を切り替えるチェックアイコンの枠付きボタンを表示する
- 記事カード右上のチェックアイコンは、未読状態では灰色、既読状態では明るい青背景に白いチェックで表示する
- ステータス切り替え後は、スナックバーに結果と「元に戻す」操作を表示する
- 記事カード上で削除操作ができ、削除前に確認ダイアログを表示する
- 記事カード上の削除アイコンは枠付きボタンとして表示する
- 検索バーの近くに並び替えセレクトを表示する
- フィルタモーダルではタグ複数選択、おすすめ度複数選択、登録日範囲、既読日範囲をまとめて変更できる
- おすすめ度フィルタは未設定を含めて左から低い順に選べる
- 登録日と既読日は、それぞれ個別に条件をクリアできる
- 追加ボタンを常時表示
- 左サイドバー下部に、画像付きの学習継続メッセージを表示する
- 学習継続メッセージは約100件のコメントと30種類程度のSVGイラストモチーフを持ち、画面遷移・記事追加・サイドメニュー選択などのアクション時にローテーションする
- 記事数が多い場合は、画面全体ではなく記事一覧領域内をスクロールする
- 記事一覧領域のスクロールバーは控えめな薄い表示にして、記事カードより強く目立たないようにする
- 記事一覧領域はスクロールバー表示の有無で記事カード幅が変化しないようにする

## カレンダー画面

- 左サイドバーにカレンダー画面への導線を表示する
- カレンダー画面の上部に表示月、前月 / 次月ボタンを表示する
- カレンダーのタイトルと登録日 / 既読日のトグルは同じ行に配置する
- 登録日 / 既読日の表示モードをトグルで切り替え、トグルは詳細画面の閲覧 / 編集トグルと同じ共通スタイルを使う
- 表示月の前月 / 次月操作は登録日 / 既読日のトグルと同じ操作行に配置し、行内中央に表示する
- カレンダー操作行の主要要素は高さを揃える
- 月内の追加数、既読数、積読差分をカレンダー上部に表示する
- 月内の追加数、既読数、積読差分はサマリー全体の囲い内で中央寄せにし、個別項目には囲いを付けず、最も長い表示に合わせた同じ最小幅で表示し、数値はラベルの右側に表示する
- 曜日見出しと日付セルは土曜、日曜で色分けする
- 日付セルは6週分の固定グリッドとして表示し、月移動や記事数の違いで高さが変わらず、カレンダー領域の下端まで揃うようにする
- 日付セルには該当日の記事数と記事タイトルを表示する
- 日付セル内の記事タイトルはクリック可能にし、記事詳細へ遷移する
- 記事数が多い日付セルはセル内でスクロールする
- カレンダー本体と日付セル内のスクロールバーは控えめな薄い表示にする

## 記事詳細画面

- 記事ヘッダーにタイトル、URL、登録日時、更新日時を表示
- URLリンクは実際のリンク先を保持したまま、表示幅を超える場合のみ見た目上 `...` で省略する
- 登録日時と更新日時はURLリンクの下に小さな補足情報として1行で表示する
- 通常は閲覧モードで表示し、戻るボタンと同じ行の中央にある閲覧 / 編集トグルで編集モードへ切り替える。トグルはカレンダー画面の表示対象切り替えと同じ共通スタイルを使う
- 閲覧モードではメイン領域に概要、タグ、メモを表示し、右側にメタ情報（ステータス、既読日、おすすめ度）を表示する
- 閲覧モードでメモが未入力の場合、メモ欄は空欄で表示する
- 閲覧モードのメモは Markdown として表示し、コードブロックはシンタックスハイライト、ファイル名表示、行番号表示に対応する
- Markdown 表示内の画像、引用、テーブル、コードブロックは詳細本文領域からはみ出さないように表示し、長いコードや表は要素内スクロールで確認できる
- 詳細画面が縦に長い場合は、閲覧 / 編集のどちらでも詳細パネル内部ではなくブラウザ画面全体でスクロールする
- 編集モードではタイトル、URL、概要、タグ、メモ、ステータス、既読日、おすすめ度を編集できる
- 右側のステータス、既読日、おすすめ度の領域は、閲覧モードと編集モードで同じサイズと位置を維持する
- 右側のステータス選択は一覧上部の並び順セレクトと同じ共通セレクトスタイルで表示する
- 右側の既読日は追加モーダルと同じ入力風UIで表示する
- 右側のおすすめ度は、閲覧モードと編集モードで星の開始位置と間隔を揃え、閲覧モードでは暗めの星で非活性表示にする
- タグ編集は追加モーダルと同様に、選択済みタグの削除、既存タグからの追加、新規タグ入力に対応する
- 右上にお気に入り、保存、削除ボタンの順で表示し、保存と削除は文字なしのアイコンボタンとして表示する
- 保存ボタンは閲覧モードでは非活性にする
- 閲覧モード中もお気に入りはヘッダーから切り替えできる
- 編集モードでは詳細ページ自体の幅を閲覧モードと揃え、モード切り替えで本文領域の幅が変わらないようにする
- 編集モードではメモ入力を本文領域の先頭に配置し、初期表示で閉じた「記事の詳細」まで画面内に収まる範囲で追記しやすい高さを確保する
- 編集モードのタイトル、URL、概要、タグ編集は、デフォルトで閉じた「記事の詳細」アコーディオンにまとめ、展開時は項目ごとに薄い横線で区切って入力フォームとして表示する
- 「記事の詳細」アコーディオンの開閉状態は、同じ詳細ページ内で閲覧 / 編集を切り替えても維持する。別画面へ移動した場合やブラウザを閉じた場合は保持しない
- 編集モードで「記事の詳細」を展開して縦に長くなる場合は、詳細パネル内ではなくブラウザ画面全体でスクロールする
- 編集モードでフォームが縦に長い場合はブラウザ画面全体でスクロールする
- 詳細編集の新しいタグ入力とメモ入力の間隔は、タイトルとURL入力の間隔と揃える
- 編集モード中に閲覧トグルへ戻すと、未保存の編集内容を破棄して閲覧モードへ戻る
- 編集モード中に未保存の差分がある状態で別画面へ移動しようとした場合は、破棄して移動するか編集を続けるかを確認する警告ダイアログを表示する
- 詳細ヘッダーのお気に入り状態のハートアイコンは赤で表示する
- 削除操作では確認ダイアログを表示し、確定後に削除する

## 追加モーダル

- URL入力欄
- モーダルを開いた直後はURL入力欄に初期フォーカスする
- URLは必須項目として扱い、未入力時は保存せずエラーメッセージを表示する
- タイトル入力欄は任意項目として表示し、未入力時はURLから自動設定されることが分かる補足を表示する
- 選択済みタグ、過去のタグからの追加、新しいタグの直接入力を分けて表示するタグ編集フォーム
- 既存タグからタグを追加した後は、既存タグ選択欄の値と検索文字列をクリアする
- おすすめ度入力を星 5 段階で表示する
- おすすめ度入力は、hover / focus中の星まで色を変えて強調し、背景色は変えない
- ステータス選択は表示しない
- 「あとで読む」チェックボックスを視認できる形で表示し、デフォルトでチェック済みにする
- 「あとで読む」チェックボックスと既読日入力は同じ行に配置し、高さと幅が揃うように表示する
- 既読日入力/日付ピッカーは追加モーダルと詳細編集で共通の見た目にし、値を上下中央、カレンダーアイコンを右端に表示する
- メモ入力欄
- 保存 / キャンセル
- キャンセルはURLバリデーションエラー表示後でもモーダルを閉じる
- 保存に失敗した場合は追加モーダル内にエラーメッセージを表示し、入力内容を保持する
- 重複 URL の場合は、登録済みの記事詳細へ移動する導線をエラーメッセージ内に表示する
- 保存後は一覧画面に留まり、自動では詳細画面へ遷移しない
