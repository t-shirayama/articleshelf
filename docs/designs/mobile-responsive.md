# スマホ対応デザイン検討

最終更新: 2026-05-07

## 1. 目的

ArticleShelf をスマホでも自然に使えるようにするため、デスクトップ版の情報設計を保ちながら、ナビゲーション、記事一覧、詳細、追加、フィルタ、カレンダーをモバイル向けに再設計する。
本ドキュメントは、役員説明では「公開時にスマホ利用をどう成立させるか」を説明する資料、開発者向けには実装に入れる詳細仕様として扱う。

## 2. 基本方針

- スマホでは記事一覧、詳細、追加、フィルタ、カレンダーを独立した画面として扱う
- 一覧と詳細の同時表示はしない
- デスクトップの左サイドバーは、スマホでは常時表示しない
- 主要移動はハンバーガーメニューとボトムナビで補助する
- 入力や確認は片手操作しやすい位置とサイズにする
- 既存のデザイン判断ルールである近接、整列、反復、対比を維持する
- モバイルでも Markdown、タグ、ステータス、日付、評価が読めることを優先する

## 3. ブレークポイント

| 幅 | 呼称 | レイアウト方針 |
| --- | --- | --- |
| `0-599px` | スマホ | ハンバーガー + ボトムナビ、1 カラム、全画面モーダル |
| `600-959px` | タブレット | 一時ドロワー、1 カラム中心、広めダイアログも可 |
| `960px以上` | デスクトップ | 左サイドバー常時表示、既存レイアウト |

Vuetify の標準ブレークポイントに合わせ、`md` 以上でデスクトップレイアウトへ戻す方針を基本にする。

## 4. ナビゲーション

### 4.1 ハンバーガーメニュー

スマホでは画面左上にハンバーガーボタンを配置し、タップでナビゲーションドロワーを開く。

ドロワー内の項目:

- 未読
- 既読
- すべて
- お気に入り
- カレンダー
- 必要に応じてログアウト、設定

仕様:

- デスクトップの左サイドバーと同じ選択状態ロジックを使う
- カレンダー画面ではカレンダーのみを選択状態にし、未読 / 既読 / お気に入りの選択色を出さない
- 背景タップ、Esc 相当、ブラウザ戻る相当で閉じられる
- 非破壊の終了操作の文言は「閉じる」
- ドロワーを閉じた後は、開いたハンバーガーボタンへ focus を戻す
- 学習継続メッセージはドロワー下部に短縮表示するか、一覧下部へ移す

### 4.2 ボトムナビ

スマホでは主要導線を下部にも置く。

推奨項目:

- 未読
- すべて
- お気に入り
- カレンダー
- 追加

仕様:

- `追加` は記事追加の主要アクションとして中央または右端に置く
- 現在画面はアイコン、ラベル、背景または上線で明確に示す
- 詳細画面では誤操作を避けるため、ボトムナビを非表示にするか、表示しても戻る導線を優先する
- 追加モーダル、削除確認、フィルタモーダル中は背後のボトムナビを操作できない
- iOS / Android の下部安全領域に対応し、`env(safe-area-inset-bottom)` を考慮する

## 5. 記事一覧

### 5.1 レイアウト

- 1 カラムの縦スクロール
- 上部に画面タイトル、検索、並び替え、フィルタ、追加導線を置く
- 記事カードは横幅いっぱいに近い幅で表示する
- OGP サムネイルは横長比率を維持し、カード上部または左寄せではなくスマホに合う大きさで表示する
- 一覧末尾がボトムナビに隠れないよう下 padding を確保する

### 5.2 カード内優先順位

1. タイトル
2. OGP サムネイル
3. ドメイン / URL
4. 概要またはメモ
5. おすすめ度、タグ、ステータス、登録日 / 既読日
6. お気に入り、既読切り替え、削除

既存仕様を維持する項目:

- タイトルは 1 行で省略
- 概要は最大 2 行
- タグは最大 3 件、残りは `+N`
- 未読 / 既読バッジを日付近くに表示
- 登録日 / 既読日は上下中央を揃える
- お気に入り状態のハートは赤
- 既読チェックは未読時灰色、既読時は明るい青背景に白チェック

### 5.3 タップ領域

- アイコンボタンは最低 44px 角を目安にする
- お気に入り、既読切り替え、削除はカード右上または下部 action row にまとめる
- 削除は誤操作を避けるため確認ダイアログを必須にする

## 6. 検索・フィルタ

### 6.1 検索バー

- 一覧上部に配置
- 横幅が狭い場合は、検索入力を 1 行目、並び替え / フィルタ / 追加を 2 行目にする
- スクロール中も条件が分からなくならないよう、適用中条件チップを検索バー付近に表示する

### 6.2 フィルタモーダル

スマホではボトムシートまたは全画面モーダルを使う。

対象:

- タグ複数選択
- おすすめ度複数選択
- 登録日範囲
- 既読日範囲

仕様:

- 条件ごとに見出しを置き、入力と補足を近接させる
- 登録日と既読日は個別にクリアできる
- おすすめ度は未設定を含めて低い順に並べる
- 適用、クリア、閉じるを明確に分ける
- 適用後は一覧へ戻り、タイトルやチップで条件が分かる

## 7. 記事詳細

### 7.1 レイアウト

- 一覧から画面遷移して表示する
- 上部に戻るボタンを置く
- 右カラムのメタ情報はスマホでは縦積みにする
- 詳細画面はブラウザ全体で縦スクロールする

### 7.2 ヘッダー

配置する操作:

- 戻る
- 閲覧 / 編集トグル
- お気に入り
- 保存
- 削除

仕様:

- 保存と削除は既存仕様どおり文字なしアイコンボタン
- 保存は閲覧モードでは非活性
- 削除は確認ダイアログを出す
- 編集中の戻る、ドロワー移動、ボトムナビ移動では未保存差分の破棄確認を出す

### 7.3 閲覧モード

表示順:

1. タイトル
2. URL
3. 登録日時 / 更新日時
4. 概要
5. タグ
6. メモ
7. ステータス、既読日、おすすめ度

Markdown:

- コード、表、画像は画面幅からはみ出さない
- 長いコードや表は要素内横スクロール
- raw HTML 無効化、DOMPurify、許可スキーム制限は維持する

### 7.4 編集モード

- メモ入力を本文領域の先頭に置く
- タイトル、URL、概要、タグは「記事の詳細」アコーディオンにまとめる
- ステータス、既読日、おすすめ度は縦積みのメタ編集ブロックにする
- 閲覧 / 編集の切り替えで画面幅が揺れないようにする
- 未保存差分がある状態で閲覧へ戻す場合は、既存仕様どおり編集内容を破棄する

## 8. 記事追加

スマホでは追加モーダルを全画面モーダルとして表示する。

### 8.1 ヘッダー

- 左: 閉じる
- 中央: タイトル
- 右: 保存する

仕様:

- 保存ボタンは常に見える位置に置く
- 閉じるは URL バリデーションエラー表示後でもフォーム送信扱いにしない
- 保存中は保存ボタンを disabled にし、二重送信を防ぐ

### 8.2 入力順

1. URL
2. タイトル
3. タグ
4. おすすめ度
5. あとで読む
6. 既読日
7. メモ

仕様:

- 初期フォーカスは URL 入力欄
- 「あとで読む」と既読日はスマホ幅では縦積みを許容する
- 各入力は十分な高さを持つ
- エラーは該当入力の近くに表示する
- 保存失敗時は入力内容を保持する
- 重複 URL 時は登録済み記事詳細への導線を表示する

## 9. カレンダー

### 9.1 月表示

- 6 週固定グリッドは維持する
- 表示月、前月 / 次月、登録日 / 既読日トグルを上部にまとめる
- 土曜 / 日曜の色分けを維持する
- 月内サマリーは追加数、既読数、積読差分を表示する

### 9.2 日付セル

スマホでは日付セル内に記事タイトルを詰め込みすぎない。

推奨:

- 日付
- 記事数
- 最大 1 件の記事タイトル
- 残りは `+N`

日付セルをタップすると、その日の記事一覧をボトムシートで表示する。
ボトムシート内の記事をタップすると詳細へ遷移する。

## 10. アクセシビリティ

- タップ領域は最低 44px 角を目安にする
- ハンバーガー、ボトムナビ、星評価、既読切り替え、削除には `aria-label` を付ける
- モーダルとドロワーは focus trap を設定する
- 閉じた後は開いたボタンへ focus を戻す
- 色だけで状態を伝えず、ラベル、アイコン、バッジ形状を併用する
- 通常テキストはコントラスト 4.5:1 以上を目安にする
- エラーは入力欄近くに表示し、スクリーンリーダーにも伝わるようにする
- キーボード操作でもドロワー、モーダル、フィルタ、削除確認を操作できる

## 11. 実装タスク

### Phase 1: レイアウト基盤

- ハンバーガードロワーを追加する
- ボトムナビを追加する
- デスクトップ sidebar とスマホ drawer の選択状態を共通化する
- safe area と下部 padding を調整する

### Phase 2: 一覧とフィルタ

- 記事カードのスマホ表示を調整する
- OGP サムネイル比率とタップ領域を確認する
- 検索 / 並び替え / フィルタ行をスマホ用に再配置する
- フィルタモーダルをスマホではボトムシートまたは全画面にする

### Phase 3: 詳細と追加

- 詳細画面の右カラム情報を縦積みにする
- 詳細ヘッダー操作をスマホ向けに整理する
- 追加モーダルを全画面化する
- 未保存差分警告が戻る / ドロワー / ボトムナビで働くようにする

### Phase 4: カレンダー

- スマホの日付セル表示を簡略化する
- 日別記事一覧ボトムシートを追加する
- 月内サマリーの折り返しを確認する

### Phase 5: 検証

- 375px、390px、600px、960px 以上で表示確認する
- Playwright でスマホ幅の smoke test を追加する
- `npm run build` を実行する
- UI 変更後は `docs/designs/screenshots/` のスクリーンショット更新を検討する

## 12. 受け入れ条件

- スマホ幅でサイドバーが常時表示されない
- ハンバーガーから未読、既読、すべて、お気に入り、カレンダーへ移動できる
- ボトムナビから主要画面と追加へ到達できる
- 記事一覧カードのタイトル、概要、タグ、日付、操作が重ならない
- スマホで記事追加を完了できる
- スマホで詳細閲覧、編集、保存、削除ができる
- 未保存差分警告がスマホ導線でも働く
- カレンダーで日付の記事を確認し、詳細へ遷移できる
- モーダルやドロワーで focus が迷子にならない
- 下部ナビで一覧末尾やボタンが隠れない
