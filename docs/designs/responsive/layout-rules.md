# Responsive Layout Rules

デスクトップ、ノート PC、タブレット、スマホにまたがる画面別レイアウトルールをまとめる。
対象サイズとブレークポイントは [README.md](README.md)、スマホ固有の詳細は [mobile.md](mobile.md) を正とする。

## 1. アプリシェル / ナビゲーション

- デスクトップでは左サイドバーを常時表示し、一覧領域と上下 padding を揃える。
- コンパクトデスクトップではサイドバー幅、シェル余白、学習継続カードの余白を縮める。
- `960px` 未満では左サイドバーを非表示にし、上部ハンバーガーと一時ドロワーに切り替える。
- `600px` 未満ではボトムナビを固定表示し、一覧末尾やボタンが隠れないよう下 padding と safe area を確保する。
- 小型スマホではブランドコピー、下部ナビラベル、アイコンボタンの間隔を詰めても、タップ領域は読み取れる状態を保つ。

## 2. 記事一覧

- デスクトップではサムネイル左、本文右の2カラムカードを維持する。
- コンパクトデスクトップでは検索・並び替え・フィルタ・追加を折り返し可能にし、各ボタンの右端や文字が切れないようにする。
- タブレット / スマホではカードを1カラム化し、OGP サムネイルは横長比率を維持しつつ過大にならない高さに抑える。
- スマホでは検索入力、並び替え、フィルタ、追加ボタンを縦積みにし、主要アクションを見失わない順序にする。
- カード内のタイトル、概要、タグ、日付、ステータス、操作ボタンは重ならないよう、狭い幅ではメタ情報を折り返す。

## 3. 記事詳細

- デスクトップでは本文と右メタカラムの2カラムを維持する。
- コンパクトデスクトップでは詳細幅とメタカラム幅を抑え、メモ編集欄の高さを画面高に合わせて調整する。
- `960px` 未満では詳細本文とメタ情報を1カラム化し、メタブロックはタブレットでは横並び、スマホでは縦積みにする。
- スマホでは戻る、閲覧 / 編集、保存、削除、お気に入りが重ならないよう、上部操作を複数行に分ける。
- 詳細画面が長くなる場合は、パネル内スクロールではなくブラウザ全体でスクロールする。

## 4. モーダル / ダイアログ

- デスクトップでは読みやすい固定幅のダイアログとして表示する。
- タブレットでは画面幅いっぱいに近い広めのダイアログを許容する。
- スマホでは記事追加、フィルタ、アカウント設定を全画面モーダルにする。
- 削除確認、タグ操作、カレンダー日別記事一覧はスマホでは下寄せシートとして表示し、下部 safe area を確保する。
- タイトル行と主要アクションは同じヘッダーに置き、本文末尾に同じ保存 / 適用操作を重複配置しない。

## 5. カレンダー / タグ管理

- カレンダーはデスクトップで6週固定グリッドを維持し、タブレット / スマホでも月移動と表示モード切り替えが重ならないようにする。
- スマホのカレンダーセルは記事タイトルを詰め込みすぎず、タップで日別記事一覧シートを開く。
- タグ管理はデスクトップでヘッダー操作群と一覧の右端を揃える。
- タブレット / スマホでは検索、並び替え、タグ追加を縦積みにし、行内編集や統合 / 削除操作が横にあふれないようにする。
