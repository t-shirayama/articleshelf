# Auth Components

AuthScreen の設計ルールをまとめる。
共通 UI ルールは [README.md](README.md) に従う。

## 1. AuthScreen

- デスクトップ幅のログイン / ユーザー登録画面は、左に ArticleShelf の価値訴求と保存・タグ・進捗の視覚カード、右に認証フォームを置く2カラム構成にする
- 1920x1080 など標準的なデスクトップ表示では1画面内に収め、フォームや視覚カードが viewport 高に収まらない短い画面や高倍率表示ではページ内スクロールを許可する
- 左カラムの視覚カードは実データではなく機能イメージとして扱い、フォーム操作や認証状態には影響させない
- タブレット幅以下ではフォーム操作を優先し、左カラムの視覚カードは非表示にしてブランドコピーとフォームを縦に並べる
- フォームカードは背景より一段明るい面、細い border、やわらかい shadow、やや大きめの角丸で自然に分離し、ヒーロー側の視覚カードより主張しすぎないコントラストに保つ
- ログイン / ユーザー登録の入力欄は、ブラウザの自動入力と重ならないよう、ラベルを入力欄の外に固定表示する
- ログイン / ユーザー登録の切り替えは、見出し直下に補助ラベル付きのセグメントコントロールとして置き、アイコン・背景・文字色で現在選択中の操作を明確に示す
- プレースホルダーは補助例示に限定し、項目名や入力ルールの代替として使わない
- ユーザー登録時のユーザー名やパスワードの入力ルールは、入力欄の直下に短いヒントとして表示する。ログイン時は既存アカウント入力を優先し、入力ルールの補足は表示しない
- ユーザー登録の表示名は任意入力とし、ラベルに任意であることを明示する。空の場合は username が表示名になることを入力欄直下のヒントで示す。表示名欄の有無でフォーム幅や送信ボタン位置を変えない
