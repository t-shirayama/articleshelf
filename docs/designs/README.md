# ArticleShelf Design

ArticleShelf のデザイン文書の入口です。
この README は共通のデザイン方針と詳細文書への導線を持ち、画面別の詳細仕様やスクリーンショット一覧は専用ファイルに分けます。

## 1. ビジュアルコンセプト

ArticleShelfのデザインは、シンプルで集中しやすいUIを目指します。
技術記事のストックを扱うため、テキストとタグが見やすいレイアウトを中心に構成します。

## 2. カラーパレット

- ベース: ホワイト / クリーム系
- アクセント: ブルーグリーン / 明るい青
- ステータス: 緑（既読） / 黄（未読） / グレー（その他）
- タグ: 淡い背景色 + 濃い文字色

## 3. タイポグラフィ

- 見出し: H1/H2 は太字で視認性を高める
- リスト項目: 行間を広くして読みやすくする
- メタ情報: 小さめフォントで補足表示
- UIフォントは `Noto Sans JP` を第一候補、`M PLUS 1p` を第二候補とし、未インストール環境では日本語OS標準ゴシックと system-ui へフォールバックする
- Markdown のコードブロックやインラインコードは `JetBrains Mono`、`Fira Code`、`Cascadia Code` を優先した等幅フォントで表示する

## 4. デザイン判断の基本4原則

- 近接: 関連する情報や操作は近づけ、別軸の情報は余白や区切り線で分ける
- 整列: サイドバー、記事一覧、詳細画面の開始位置や主要コンテンツの端を揃える
- 反復: ボタン、カード、入力欄、余白、角丸、アイコンサイズは共通パターンを繰り返す
- 対比: 主要アクション、状態、エラー、選択中項目は色・太さ・背景で明確に差をつける
- UI 調整後は、セレクト値の見切れ、入力欄とボタンの左右端の不揃い、状態変化時の高さ変化、長い日本語/英語文言の収まりを確認する

## 6. 基本ユーザーフロー

1. ブラウザから記事を見つける
2. 記事URLをコピーして ArticleShelf に貼り付け
3. タイトルとタグ、既読日を入力して保存
4. 記事一覧で状態を確認
5. 記事追加後は一覧画面に留まり、必要な記事を選択したときだけ詳細画面へ遷移
6. 既読後や振り返り時にメモを追記
