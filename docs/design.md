# ReadStack Design

## 1. ビジュアルコンセプト

ReadStackのデザインは、シンプルで集中しやすいUIを目指します。
技術記事のストックを扱うため、テキストとタグが見やすいレイアウトを中心に構成します。

## 2. カラーパレット

- ベース: ホワイト / クリーム系
- アクセント: ブルー / パープル
- ステータス: 緑（読了） / 黄（未読） / グレー（その他）
- タグ: 淡い背景色 + 濃い文字色

## 3. タイポグラフィ

- 見出し: H1/H2 は太字で視認性を高める
- リスト項目: 行間を広くして読みやすくする
- メタ情報: 小さめフォントで補足表示

## 4. レイアウト

### 4.0 デザイン判断の基本4原則

- 近接: 関連する情報や操作は近づけ、別軸の情報は余白や区切り線で分ける
- 整列: サイドバー、記事一覧、詳細画面の開始位置や主要コンテンツの端を揃える
- 反復: ボタン、カード、入力欄、余白、角丸、アイコンサイズは共通パターンを繰り返す
- 対比: 主要アクション、状態、エラー、選択中項目は色・太さ・背景で明確に差をつける

### 4.1 PC / デスクトップ

- 左サイドバー: フィルター / タグ / ナビゲーション
- 左サイドバーと中央メインの上下 padding は揃え、画面内の開始位置を整列させる
- 左サイドバー下部: 画像付きの学習継続メッセージ
- 学習継続メッセージは約100件のコピーと30種類程度のSVGイラストモチーフを持ち、画面遷移・記事追加・サイドメニュー選択時に切り替える
- 中央メイン: 記事一覧を表示
- 一覧画面のタイトルは、左サイドバーで選択中の項目（すべての記事、未読、読了、お気に入り、タグ名）に合わせて表示する
- 記事を選択したら、一覧と同時表示せず詳細画面へ遷移
- 詳細画面では、本文領域とメタ情報 / 操作ボタンを分けて表示
- 記事やタグが増えても画面全体ではなく、記事一覧領域・タグ領域の中でスクロールする

### 4.2 モバイル

- ハンバーガーメニュー / ボトムナビゲーション
- 記事一覧を縦スクロールで表示
- 詳細は画面遷移で表示し、戻るボタンで一覧へ戻る

## 5. コンポーネント設計

UIの基本部品には Vuetify を使用します。
ボタン、入力欄、カード、ダイアログ、チップ、日付入力などは Vuetify を優先し、ReadStack固有の余白・背景・レイアウトはカスタムCSSで調整します。
保存、追加、編集、キャンセル、削除などの主要アクションは、押せる領域が明確に見える共通ボタンスタイルを使います。

### 5.1 ArticleCard

- タイトル
- URL / ドメイン
- タグ
- 読了ステータスバッジ
- おすすめ度の星表示
- 登録日 / 読了日はおすすめ度と同じ行の右端に高さを揃えて表示する
- お気に入り追加 / 解除ボタン
- お気に入り追加 / 解除は一覧全体をローディング表示に戻さず、カード上で即時反映して裏で保存する
- OGP画像はQiitaやZennなどの横長OGPを見やすいサムネイルとして表示し、サムネイル領域が表示範囲に近づいた時だけ取得する。取得した画像はブラウザのIndexedDBに保存した画像Blobを表示し、未保存または取得不可の場合はプレースホルダーを表示する

### 5.2 ArticleDetail

- タイトル + URLリンク
- URLリンクは表示領域いっぱいまで使い、収まりきらない場合のみ見た目上 `...` で省略する
- 通常は閲覧モードで表示し、編集ボタン押下時のみ編集モードへ切り替える
- 閲覧モードでは概要、タグ、メモを読みやすい表示で見せる
- 右カラムにステータス、読了日、登録日時、更新日時、おすすめ度を表示する
- 右カラムのステータス、読了日、登録日時、更新日時、おすすめ度、削除は固定スロットとして反復し、閲覧 / 編集の切り替えでサイズや位置を変えない
- 右カラムのステータス選択は一覧上部の並び順セレクトと同じ共通セレクトスタイルを使い、画面間の操作感を揃える
- 右カラムの読了日、登録日時、更新日時は同じ入力風UIで表示し、登録日時と更新日時は閲覧・編集どちらでも時刻つきで非活性表示する
- 右カラムのおすすめ度は閲覧 / 編集で星の開始位置と間隔を揃え、閲覧時は星を暗めにして非活性を表現する
- 既存タグ選択 + 新規タグ入力のタグ編集は編集モードで行う
- favorite は詳細ヘッダーから切り替えできる
- 編集モードでは保存 / キャンセルを表示する
- 削除ボタン押下時の確認ダイアログ

### 5.3 ArticleFormModal

- Vuetify のダイアログとして表示
- 入力フィールド: URL, タイトル, タグ, おすすめ度, あとで読む, 読了日, メモ
- モーダルを開いた直後はURL入力欄に初期フォーカスを置き、コピーした記事URLをすぐ貼り付けられるようにする
- フォーム内の各入力ブロックは同じ縦余白で反復し、補足文だけを入力欄に近接させる
- フォーム本文とアクション行の左右余白を揃え、保存 / キャンセルのボタン位置を整列させる
- URLは必須項目として扱い、未入力時はエラーを表示
- タイトルは任意項目として扱い、未入力時はURLから自動設定されることを補足表示
- タグ入力は、過去のタグからの選択と新規入力の両方に対応
- おすすめ度はhover / focus中に対象の星まで背景色を変え、保存される段階が視覚的に分かるようにする
- ステータス選択は表示せず、視認しやすい「あとで読む」チェックボックスをデフォルトでチェック済みにする
- 「あとで読む」と読了日は同じ行に配置し、高さと幅を揃える
- 読了日入力は追加モーダルと詳細編集で共通スタイルを使い、値を上下中央、カレンダーアイコンを右端に揃える
- アクション: 保存 / キャンセル
- バリデーションメッセージ表示

### 5.4 SearchFilterBar

- 全文検索入力
- 並び替えセレクト
- 並び替えセレクトは詳細画面のステータス選択と同じ共通セレクトスタイルを使う
- 追加ボタン
- 未読 / 読了 / すべての絞り込みは左サイドバーに集約し、SearchFilterBar には重複するステータストグルを表示しない

## 6. ユーザーフロー

1. ブラウザから記事を見つける
2. 記事URLをコピーして ReadStack に貼り付け
3. タイトルとタグ、読了日を入力して保存
4. 記事一覧で状態を確認
5. 記事追加後は一覧画面に留まり、必要な記事を選択したときだけ詳細画面へ遷移
6. 読了後や振り返り時にメモを追記

## 7. モックアップ想定

実装の参考となるデザイン画面は `docs/designs/` に格納しています。

- `desktop_article_list.png`: デスクトップ版 記事一覧画面
  - 左サイドバーでフィルターを選択
  - 左サイドバー下部に学習継続を後押しする画像付きメッセージを表示
  - メッセージは固定せず、アクションに合わせてローテーション表示
  - 中央にカード形式で記事を表示
  - 各カードにサムネイル、タイトル、タグ、ステータス、お気に入り操作を表示

- `desktop_article_detail_light.png`: デスクトップ版 記事詳細画面
  - ダークテーマで落ち着いた見た目
  - タイトル、URL、タグを上部に配置
  - 記事イメージ、概要を中央に表示
  - 右側に読了日、追加日などのメタ情報
  - メモを下部に表示・編集可能

- `add_article_modal.png`: 記事追加モーダル
  - URL、タイトル、タグ、あとで読む、読了日、メモの入力フィールド
  - タグは既存タグ選択と新規入力に対応
  - キャンセル・保存ボタン

- `mobile_article_list.png`: モバイル版 記事一覧画面
  - 画面幅に合わせたカード表示
  - フローティングアクションボタンで記事追加
  - ハンバーガーメニューまたはボトムナビゲーション対応

## 8. 拡張デザイン要件

- 画像添付用のサムネイル表示
- お気に入り / ピン留めの視覚表現
- タグのバッジ化による視認性向上
- 夜間モード / ダークテーマ対応
