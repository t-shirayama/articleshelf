# ReadStack Design

## 1. ビジュアルコンセプト

ReadStackのデザインは、シンプルで集中しやすいUIを目指します。
技術記事のストックを扱うため、テキストとタグが見やすいレイアウトを中心に構成します。

## 2. カラーパレット

- ベース: ホワイト / クリーム系
- アクセント: ブルーグリーン / 明るい青
- ステータス: 緑（既読） / 黄（未読） / グレー（その他）
- タグ: 淡い背景色 + 濃い文字色

## 3. タイポグラフィ

- 見出し: H1/H2 は太字で視認性を高める
- リスト項目: 行間を広くして読みやすくする
- メタ情報: 小さめフォントで補足表示
- UIフォントは `Noto Sans JP` を第一候補、`M PLUS 1p` を第二候補とし、未インストール環境では日本語OS標準ゴシックと system-ui へフォールバックする
- Markdown のコードブロックやインラインコードは `JetBrains Mono`、`Fira Code`、`Cascadia Code` を優先した等幅フォントで表示する

## 4. レイアウト

### 4.0 デザイン判断の基本4原則

- 近接: 関連する情報や操作は近づけ、別軸の情報は余白や区切り線で分ける
- 整列: サイドバー、記事一覧、詳細画面の開始位置や主要コンテンツの端を揃える
- 反復: ボタン、カード、入力欄、余白、角丸、アイコンサイズは共通パターンを繰り返す
- 対比: 主要アクション、状態、エラー、選択中項目は色・太さ・背景で明確に差をつける
- UI 調整後は、セレクト値の見切れ、入力欄とボタンの左右端の不揃い、状態変化時の高さ変化、長い日本語/英語文言の収まりを確認する

### 4.1 PC / デスクトップ

- 左サイドバー: ステータス / お気に入り / カレンダーのナビゲーション
- アプリのファビコンは左サイドバー左上のブランドマークに合わせる
- 左サイドバーの選択表示は現在表示中の画面を優先し、カレンダー画面では未読 / 既読 / お気に入りなど一覧フィルタの選択色を出さない
- 左サイドバーと中央メインの上下 padding は揃え、画面内の開始位置を整列させる
- 左サイドバー下部: 画像付きの学習継続メッセージ
- 学習継続メッセージは約100件のコピーと30種類程度のSVGイラストモチーフを持ち、画面遷移・記事追加・サイドメニュー選択時に切り替える
- ログアウトは左サイドバー下部に枠付きボタンとして表示し、通常ナビゲーションやユーザー名と区別できるようにする
- 左サイドバー下部にはユーザー名、その下に日本語 / English の言語切替トグル、ログアウトボタンの順で表示する。言語切替はアカウント単位ではなく端末側の表示設定として扱う
- 中央メイン: 記事一覧を表示
- 一覧画面のタイトルは、左サイドバーで選択中の項目や上部フィルタ条件に合わせて表示する
- カレンダー画面では、登録日と既読日の表示モードを切り替え、月内の追加数・既読数・積読差分を表示する
- 記事を選択したら、一覧と同時表示せず詳細画面へ遷移
- 詳細画面では、本文領域とメタ情報 / 操作ボタンを分けて表示
- 記事が増えても画面全体ではなく、記事一覧領域の中でスクロールする
- 記事一覧やカレンダー内のスクロールバーは、カードより目立たない薄いトーンで表示する
- 記事一覧領域はスクロールバーの有無で記事カード幅が変わらないよう、スクロールバー用の余白を安定して確保する

### 4.2 モバイル

- ハンバーガーメニュー / ボトムナビゲーション
- 記事一覧を縦スクロールで表示
- 詳細は画面遷移で表示し、戻るボタンで一覧へ戻る
- 詳細なスマホ対応方針、ブレークポイント、各画面の実装タスクは `design/mobile-responsive.md` を正とする

## 5. コンポーネント設計

UIの基本部品には Vuetify を使用します。
ボタン、入力欄、カード、ダイアログ、チップ、日付入力などは Vuetify を優先し、ReadStack固有の余白・背景・レイアウトはカスタムCSSで調整します。
保存、追加、編集、閉じる、キャンセル、削除などの主要アクションは、押せる領域が明確に見える共通ボタンスタイルを使います。
非破壊のモーダルを閉じる操作は「閉じる」、削除確認など進行中の確認を取りやめる操作は「キャンセル」と表記します。
Vuetify のUIロケールは日本語を標準とし、カレンダーや入力補助UIも日本語表示にします。
日付を画面に表示する場合は `YYYY/MM/DD` のスラッシュ区切りを共通表記とし、APIや永続化で扱う日付値は既存仕様に合わせて `YYYY-MM-DD` を維持します。
日付ピッカーは一般的な日本語カレンダーに合わせ、上部は年・月の順で表示し、日曜は赤系、土曜は青系で見分けられるようにします。

### 5.0 モーダル共通ルール

- 新規作成・編集・絞り込みなど操作完了を伴うモーダルは、タイトルと主要アクションを同じヘッダー行に配置する
- ヘッダー行の左側にタイトル、右側に「閉じる」と保存・適用などの主要アクションを並べ、本文末尾に同じアクションを重複配置しない
- 条件クリアやリセットなど補助アクションがある場合も、文脈上ヘッダー行に収まるものはタイトル行にまとめる
- フォーム本文とヘッダー行の左右余白を揃え、入力ブロックは薄い横線と一定の縦余白で反復する
- モバイルではヘッダーを縦積みにし、アクションボタンを横幅いっぱいにして押しやすさを優先する

### 5.1 ArticleCard

- タイトル
- 記事カード内ではタイトルを最も強い視覚要素として扱い、ドメインや概要より大きめ・太め・高コントラストで表示する
- 記事カードのタイトルは1行で表示し、長い場合は末尾を省略してカード内の他要素と重ならないようにする
- 記事カードの概要は2行分の固定領域に収め、タグや日付を含むメタ行と重ならないようにする
- 記事カードの概要やメモが未入力の場合、代替文言は表示せず空欄にする
- URL / ドメイン
- タグ
- 既読ステータスバッジ
- 記事カード上で未読 / 既読を切り替える操作は右上のアクション群にチェックアイコンの枠付きボタンとして配置する
- 記事カード右上のチェックアイコンは未読状態では灰色、既読状態では明るい青背景に白いチェックで表示する
- ステータス切り替え後はスナックバーで元に戻す導線を表示する
- おすすめ度の星表示
- タグはおすすめ度の横に配置する
- 記事カード上のタグは最大3件まで表示し、残りは `+N` のチップにまとめて情報密度を抑える
- 未読 / 既読ステータスバッジは登録日 / 既読日のすぐ左に配置し、おすすめ度・タグ・日付と同じ行で表示する
- 登録日 / 既読日はおすすめ度・タグ・ステータスと同じ行の右端に高さを揃えて表示する
- 記事カード上の登録日 / 既読日は同じ高さの表示枠で上下中央を揃え、既読日が未設定の場合は「未設定」と表示する
- 記事カード上の登録日 / 既読日は長い方に合わせた同じ最小幅で表示する
- お気に入り追加 / 解除ボタン
- お気に入り状態のハートは赤で表示する
- お気に入り追加 / 解除は一覧全体をローディング表示に戻さず、カード上で即時反映して裏で保存する
- 記事カード上の削除アイコンは枠付きボタンとして表示し、クリック可能な操作であることを明確にする
- OGP画像はQiitaやZennなどの横長OGPを見やすい大きめのサムネイルとして表示し、サムネイル領域が表示範囲に近づいた時だけ取得する。取得した画像はブラウザのIndexedDBに保存した画像Blobを表示し、未保存または取得不可の場合はプレースホルダーを表示する

### 5.2 ArticleDetail

- タイトル + URLリンク
- URLリンクは表示領域いっぱいまで使い、収まりきらない場合のみ見た目上 `...` で省略する
- 登録日時と更新日時はURLリンクの下に小さな補足情報として1行で表示する
- 通常は閲覧モードで表示し、戻るボタンと同じ行の中央にある閲覧 / 編集トグルで編集モードへ切り替える。トグルはカレンダー画面の表示対象切り替えと同じ共通スタイルを使う
- 閲覧モードでは概要、タグ、メモを読みやすい表示で見せる
- 閲覧モードでメモが未入力の場合、代替文言は表示せず空欄にする
- 閲覧モードのメモは Markdown として表示し、コードブロックはシンタックスハイライト、ファイル名表示、行番号表示に対応する
- Markdown 表示内の画像、引用、テーブル、コードブロックは詳細本文領域からはみ出さないように収め、必要な横方向の長さは要素内スクロールで扱う
- 編集モードのメモ入力では、メモが検索対象になることと Markdown 記法を使えることを補足表示する
- 詳細画面が縦に長い場合は、閲覧 / 編集のどちらでも詳細パネル内部ではなくブラウザ画面全体でスクロールする
- 右カラムにステータス、既読日、おすすめ度を表示する
- 右カラムのステータス、既読日、おすすめ度は固定スロットとして反復し、閲覧 / 編集の切り替えでサイズや位置を変えない
- 右カラムのステータス選択は一覧上部の並び順セレクトと同じ共通セレクトスタイルを使い、画面間の操作感を揃える
- 右カラムの既読日は追加モーダルと同じ入力風UIで表示する
- 右カラムのおすすめ度は閲覧 / 編集で星の開始位置と間隔を揃え、閲覧時は星を暗めにして非活性を表現する
- タグ編集は、選択済みタグの削除、既存タグからの追加、新規タグ入力を分けて編集モードで行う
- 既存タグからタグを追加した後は、既存タグ選択欄の値と検索文字列をクリアする
- favorite は詳細ヘッダーから切り替えでき、お気に入り状態のハートは赤で表示する
- 右上にお気に入り、保存、削除ボタンの順で表示し、保存と削除は文字なしのアイコンボタンにする
- 保存ボタンは閲覧モードでは非活性にする
- 編集モードでは詳細ページ自体の幅を閲覧モードと揃え、モード切り替えで本文領域がちらつかないようにする
- 編集モードではメモ入力を本文領域の先頭に配置し、詳細画面を開いてすぐ追記できる導線にする
- 編集モードの初期表示では、メモ入力と閉じた「記事の詳細」までを画面内に収め、メモ入力は振り返りや追記に使いやすい十分な高さを確保する
- 編集モードのタイトル、URL、概要、タグ編集はデフォルトで閉じた「記事の詳細」アコーディオンにまとめ、展開時は項目ごとに薄い横線と線前後の余白で区切る。タグ編集は高さを取るため、小見出しで入力領域の始まりを示す
- 「記事の詳細」アコーディオンの開閉状態は、同じ詳細ページ内で閲覧 / 編集を切り替えても維持し、別画面遷移や再読み込みでは破棄する
- 編集モードで「記事の詳細」を展開して縦に長くなる場合は、詳細パネル内ではなくブラウザ画面全体でスクロールする
- 編集モードのタイトル、URL、概要は入力フォームとして表示し、タグ編集UIは既存のタグ編集パターンを流用する
- 編集モードで縦に長いフォームはブラウザ画面全体でスクロールする
- 詳細編集の新しいタグ入力とメモ入力の間隔は、タイトルとURL入力の間隔と揃える
- 編集モードで閲覧トグルへ戻すと編集内容を破棄して閲覧モードへ戻る
- 編集モードで未保存の差分がある状態で一覧など別画面へ移動しようとした場合は、破棄して移動するか編集を続けるかを確認する警告ダイアログを表示する
- 削除ボタン押下時の確認ダイアログ

### 5.3 ArticleFormModal

- Vuetify のダイアログとして表示
- 入力欄が窮屈に見えないよう、記事追加モーダルはフォームを読みやすい広めの幅で表示する
- タイトル、閉じる、保存するボタンは同じヘッダー行に配置し、タイトル下の余白を抑える
- 入力フィールド: URL, タイトル, タグ, おすすめ度, あとで読む, 既読日, メモ
- モーダルを開いた直後はURL入力欄に初期フォーカスを置き、コピーした記事URLをすぐ貼り付けられるようにする
- フォーム内の各入力ブロックは薄い横線と同じ縦余白で反復し、補足文だけを入力欄に近接させる
- フォーム本文とヘッダー行の左右余白を揃え、保存 / 閉じるのボタン位置を整列させる
- URLは必須項目として扱い、未入力時はエラーを表示
- タイトルは任意項目として扱い、未入力時はURLから自動設定されることを補足表示
- タグ入力は小見出しを添え、選択済みタグ、既存タグからの追加、新規タグ入力を分けて表示する
- タグ入力の選択済みタグ領域は、未選択時の空状態とタグ追加後の1行表示で高さを揃え、タグ追加時にモーダル全体がちらつかないようにする
- 既存タグからタグを追加した後は、既存タグ選択欄の値と検索文字列をクリアする
- おすすめ度はタグ入力と同じ小見出しを添え、hover / focus中に対象の星まで色を変え、背景色は変えない
- ステータス選択は表示せず、視認しやすい「あとで読む」チェックボックスをデフォルトでチェック済みにする
- 「あとで読む」と既読日は同じ行に配置し、高さと幅を揃える
- 既読日入力は追加モーダルと詳細編集で共通スタイルを使い、値を上下中央に揃え、右端のカレンダーアイコンボタンから Vuetify の日付ピッカーを開けるようにする
- アクション: ヘッダー右側に保存 / 閉じるを配置し、本文末尾には重複配置しない
- 閉じるはURLバリデーションエラー表示後でもフォーム送信扱いにせず、常にモーダルを閉じる
- バリデーションメッセージ表示

### 5.4 SearchFilterBar

- 全文検索入力
- 並び替えセレクト
- フィルタモーダルを開くボタン
- フィルタボタン付近に適用中条件の要約チップを表示する
- 並び替えセレクトは詳細画面のステータス選択と同じ共通セレクトスタイルを使う
- 追加ボタン
- 未読 / 既読 / すべての絞り込みは左サイドバーに集約し、SearchFilterBar には重複するステータストグルを表示しない
- タグ、おすすめ度、登録日範囲、既読日範囲はフィルタモーダルでまとめて変更できる
- フィルタモーダルのタグ選択は記事追加モーダルのタグUIを流用し、選択済みタグと既存タグからの追加を分けて表示する。フィルタでは新規タグ追加欄は表示しない
- フィルタモーダルの日付範囲入力も既読日入力と同じ Vuetify 日付ピッカー付きの共通日付入力を使う
- フィルタモーダルの条件クリア、閉じる、適用するボタンは、記事追加モーダルと同様にタイトル行の右側へ配置する

### 5.5 CalendarView

- 登録日 / 既読日の表示モード切り替えは、詳細画面の閲覧 / 編集トグルと同じ共通スタイルを使う
- カレンダーのタイトルと登録日 / 既読日のトグルは同じ操作行に置き、縦方向の占有を減らしてカレンダー本体を広く取る
- 前月 / 次月の月移動
- 表示モード切り替えと月移動は同じ操作行に配置し、月移動は中央に揃える
- 月内の追加数、既読数、積読差分
- 月内サマリーの追加数、既読数、積読差分はサマリー全体の囲い内で中央寄せにし、個別項目には囲いを付けず、同じ最小幅で表示し、数値はラベルの右側に置く
- 土曜と日曜は一般的なカレンダーと同様に色分けする
- 日付セルは6週分の固定グリッドで表示し、月移動や記事数で高さを揺らさず、カレンダー領域の下端まで揃える
- 日付ごとの記事タイトル一覧
- 記事タイトルクリックで記事詳細へ遷移

### 5.6 TagManagementView

- 左サイドバーからタグ管理画面へ移動できる
- タグ管理画面ではタグ名、紐づく記事数、リネーム、統合、未使用タグ削除を一覧で扱う
- タグ管理画面の見出し横に登録タグ数を `N件` で表示する
- タグ名検索、記事数が多い順 / 少ない順 / 名前順の並び替え、タグ追加ボタンをタグ管理タイトルと同じヘッダー行に配置し、記事一覧画面の上部操作と同じ見え方に揃える
- 並び替えはラベル付きのセレクトとして表示し、何の選択欄か分かるようにする
- タグ管理の並び替えセレクトは日本語/英語の選択値が見切れない幅を確保する
- タグ追加は「タグを追加」ボタンからダイアログを開き、タグ名を入力して追加する
- タグ追加ダイアログではタグ名入力欄の右端と追加ボタンの右端を揃える
- リネームは行内編集で行い、保存とキャンセルはアイコンボタンで表示する
- 編集、統合、削除の操作はテキスト付きボタンで表示し、ツールチップで意味を補足する
- 統合は確認ダイアログで統合先タグを選択し、元タグの記事紐づけを統合先へ移す
- 削除は未使用タグのみ操作可能にし、使用中タグは削除ボタンを非活性にし、ツールチップで「使用中のタグは削除できません」と表示する
- 未使用タグ削除の確認文では、記事自体は削除されないことを明記する
- タグ名または記事数を選択すると、そのタグで絞り込んだ記事一覧へ移動する
- タグが0件の場合は、タグを先に用意できることを案内し、最初のタグを追加する導線を表示する

### 5.7 AuthScreen

- ログイン / ユーザー登録の入力欄は、ブラウザの自動入力と重ならないよう、ラベルを入力欄の外に固定表示する
- プレースホルダーは補助例示に限定し、項目名の代替として使わない

## 6. ユーザーフロー

1. ブラウザから記事を見つける
2. 記事URLをコピーして ReadStack に貼り付け
3. タイトルとタグ、既読日を入力して保存
4. 記事一覧で状態を確認
5. 記事追加後は一覧画面に留まり、必要な記事を選択したときだけ詳細画面へ遷移
6. 既読後や振り返り時にメモを追記

## 7. 現行画面キャプチャ

`docs/designs/` には、現行実装から取得した代表スクリーンショットを格納しています。
差し替え前の画像は `docs/designs/archive/` に退避し、現在の説明に使う画像はルート直下のファイルを正とします。
デスクトップ版のキャプチャは `1920x1080`、ブラウザ locale は `ja-JP` で取得します。
キャプチャスクリプトは専用ユーザーと記事/タグを作成してから撮影し、既存ログイン状態や手元データへ依存しないようにします。

- `auth_login.png`: ログイン画面
  - ブラウザ自動入力とラベルが重ならない固定ラベル形式
  - ログイン / 登録の切り替えと主要アクション

- `desktop_article_list.png`: デスクトップ版 記事一覧画面
  - 左サイドバーで未読 / 既読 / お気に入りなどの状態を選択
  - 上部のフィルタボタンからタグ、おすすめ度、日付条件を絞り込む
  - 左サイドバー下部に学習継続を後押しする画像付きメッセージを表示
  - メッセージは固定せず、アクションに合わせてローテーション表示
  - 中央にカード形式で記事を表示
  - 各カードにサムネイル、タイトル、タグ、ステータス、お気に入り操作を表示

- `desktop_article_detail_light.png`: デスクトップ版 記事詳細画面
  - 現在の実装はクリーム系のライトテーマで統一
  - タイトル、URL、タグを上部に配置
  - 記事イメージ、概要を中央に表示
  - 右側に既読日、登録日などのメタ情報
  - メモを下部に表示・編集可能

- `calendar_view.png`: カレンダー画面
  - 登録日 / 既読日の表示モード切り替え
  - 月移動、月内サマリー、日付ごとの記事表示

- `tag_management.png`: タグ管理画面
  - タグ名検索、並び替え、タグ追加ボタンをヘッダー行に配置
  - タグ名、記事数、編集、統合、削除の操作を一覧で表示

- `add_article_modal.png`: 記事追加モーダル
  - URL、タイトル、タグ、あとで読む、既読日、メモの入力フィールド
  - タグは選択済みタグ、既存タグからの追加、新規タグ入力に対応
  - 閉じる・保存ボタン

- `filter_dialog.png`: フィルタモーダル
  - タグ、おすすめ度、登録日範囲、既読日範囲をまとめて調整
  - 条件クリア、閉じる、適用する操作をヘッダー行に配置

- `tag_add_dialog.png`: タグ追加モーダル
  - タグ名を単体で追加するフォーム
  - 入力欄と追加ボタンの右端を揃える

- `tag_merge_dialog.png`: タグ統合モーダル
  - 統合元タグと統合先セレクトを表示
  - 統合後の記事紐づけ移動を確認して実行

- `tag_delete_dialog.png`: タグ削除確認ダイアログ
  - 未使用タグの削除対象を確認する
  - 記事自体は削除されないことを文言で補足する

- `delete_article_dialog.png`: 記事削除確認ダイアログ
  - 削除対象を確認し、キャンセル / 削除するを明確に分ける

- `mobile_article_list.png`: モバイル版 記事一覧画面
  - 画面幅に合わせたカード表示
  - 現在の実装では、サイドバーと一覧が縦積みされるレスポンシブ構成を確認できる
