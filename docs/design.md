# ReadStack Design

## 1. ビジュアルコンセプト

ReadStackのデザインは、シンプルで集中しやすいUIを目指します。
技術記事のストックを扱うため、テキストとタグが見やすいレイアウトを中心に構成します。

## 2. カラーパレット

- ベース: ホワイト / クリーム系
- アクセント: ブルー / パープル
- ステータス: 緑（読了） / 黄（未読） / グレー（その他）
- タグ: 淡い背景色 + 濃い文字色

## 3. タイポグラフィ

- 見出し: H1/H2 は太字で視認性を高める
- リスト項目: 行間を広くして読みやすくする
- メタ情報: 小さめフォントで補足表示

## 4. レイアウト

### 4.1 PC / デスクトップ

- 左サイドバー: フィルター / タグ / ナビゲーション
- 中央メイン: 記事一覧 or 詳細表示
- 右パネル: 記事のメタ情報 / 操作ボタン

### 4.2 モバイル

- ハンバーガーメニュー / ボトムナビゲーション
- 記事一覧を縦スクロールで表示
- 詳細はフルスクリーンモーダルまたは画面遷移

## 5. コンポーネント設計

### 5.1 ArticleCard

- タイトル
- URL / ドメイン
- タグ
- 読了ステータスバッジ
- 読了日 / 追加日

### 5.2 ArticleDetail

- タイトル + URLリンク
- タグリスト
- ステータス切り替えボタン
- 読了日 / favorite / メモ
- 編集ボタン / 保存ボタン

### 5.3 ArticleFormModal

- 入力フィールド: URL, タイトル, タグ, 読了日, メモ
- アクション: 保存 / キャンセル
- バリデーションメッセージ表示

### 5.4 SearchFilterBar

- 全文検索入力
- ステータスタブ
- タグ選択ドロップダウン
- 追加ボタン

## 6. ユーザーフロー

1. ブラウザから記事を見つける
2. 記事URLをコピーして ReadStack に貼り付け
3. タイトルとタグ、読了日を入力して保存
4. 記事一覧で状態を確認
5. 読了後や振り返り時にメモを追記

## 7. モックアップ想定

実装の参考となるデザイン画面は `docs/designs/` に格納しています。

- `desktop_article_list.png`: デスクトップ版 記事一覧画面
  - 左サイドバーでフィルターを選択
  - 中央にカード形式で記事を表示
  - 各カードにサムネイル、タイトル、タグ、ステータスを表示

- `desktop_article_detail_light.png`: デスクトップ版 記事詳細画面
  - ダークテーマで落ち着いた見た目
  - タイトル、URL、タグを上部に配置
  - 記事イメージ、概要を中央に表示
  - 右側に読了日、追加日などのメタ情報
  - メモを下部に表示・編集可能

- `add_article_modal.png`: 記事追加モーダル
  - URL、タイトル、タグ、メモの入力フィールド
  - キャンセル・保存ボタン

- `mobile_article_list.png`: モバイル版 記事一覧画面
  - 画面幅に合わせたカード表示
  - フローティングアクションボタンで記事追加
  - ハンバーガーメニューまたはボトムナビゲーション対応

## 8. 拡張デザイン要件

- 画像添付用のサムネイル表示
- お気に入り / ピン留めの視覚表現
- タグのバッジ化による視認性向上
- 夜間モード / ダークテーマ対応
