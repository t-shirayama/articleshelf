# PDF インポート品質要件

PDF インポートは、URL 直接登録よりも抽出結果が揺れやすい入力導線として扱う。
現時点では実装前の品質要件を定義し、実装時はここで定義した補正フローと確認ケースを仕様・デザインへ昇格する。

## 1. 抽出失敗パターン

- PDF 内に URL がない、または本文中に複数 URL があり候補を一意にできない
- タイトルがヘッダー / フッター / ページ番号 / 著者名と混ざる
- 改行、ハイフネーション、段組により概要やメモ候補が不自然につながる
- スキャン PDF などでテキスト抽出できない
- 日本語 / 英語が混在し、タグ候補や要約候補の言語判定が揺れる
- URL は抽出できるが、OGP 取得失敗や SSRF guard により補完できない

## 2. 補正フロー

- 抽出結果は自動保存せず、URL、タイトル、概要、タグ、メモ候補を確認できる編集画面へ渡す
- URL が複数ある場合は候補リストから選択できるようにし、確信できない候補を勝手に採用しない
- URL、タイトル、概要、タグ、メモ候補の欠落や低信頼度を項目ごとに表示する
- 手動補正後の保存は通常の記事追加と同じ validation、重複 URL、OGP 補完、タグ正規化を通す
- テキスト抽出できない PDF では、ファイル内容を保存対象にせず、URL 手入力へ誘導する

## 3. エラー表示方針

- 抽出不能、候補過多、URL 不正、OGP 取得失敗を区別して表示する
- ユーザーに見せる文言は「何を直せば保存できるか」を中心にし、PDF parser や OCR の内部エラーは露出しない
- 低信頼度でも保存可能な項目は警告として扱い、必須項目の URL がない場合だけ保存を止める

## 4. 代表確認ケース

| ケース | 期待結果 |
| --- | --- |
| URL とタイトルが1つずつ明確な PDF | URL / タイトルが入力欄に入り、ユーザーが確認して保存できる |
| URL が複数ある PDF | URL 候補を選択でき、未選択のまま保存できない |
| URL がない PDF | URL 手入力を促し、自動保存しない |
| 段組 / 改行が多い PDF | 概要 / メモ候補が低信頼度として表示され、手動補正できる |
| スキャン PDF | テキスト抽出不能として扱い、内部例外を表示しない |
| OGP 取得に失敗する URL | 通常の記事追加と同じ URL unavailable の扱いにする |
