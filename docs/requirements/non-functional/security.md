# セキュリティの非機能要件

ArticleShelf はユーザーごとの記事、タグ、メモ、既読履歴を扱うため、認証、認可、入力検証、外部通信、表示安全性を非機能要件として扱います。
現在の実装仕様は [../../specs/security/README.md](../../specs/security/README.md) を正とします。

## 要件

- 保護 API は認証済みユーザーだけが利用できること
- 記事、タグ、メモ、既読履歴はユーザーごとに分離され、他ユーザーのデータを参照・更新・削除できないこと
- セッション継続に使う token は漏えい時の影響を抑えられるよう、短命 access token と安全な refresh token 管理に分けること
- refresh token を cookie で扱う API は CSRF から保護すること
- 公開環境では CORS origin、cookie secure 属性、secret、DB TLS を明示し、安全でない既定値のまま起動しないこと
- 登録 / ログインなどの公開 API は過剰試行を抑止できること
- ユーザー入力 URL を backend が取得する処理は SSRF を防止すること
- 入力不正、認証失敗、権限不足、重複、想定外エラーは内部詳細を漏らさず、利用者が理解できる形で返すこと
- Markdown メモはユーザー入力として扱い、script 実行、危険タグ、危険属性、危険 scheme を表示・実行しないこと
- セキュリティ境界は自動テストで回帰確認できること
