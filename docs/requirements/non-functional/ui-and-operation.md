# 画面・運用の非機能要件

- デスクトップ、ノート PC、タブレット、スマホで主要導線を操作できるレスポンシブ UI を維持する
- データ永続化を前提とした信頼性を確保する
- 将来的な拡張を見据えた設計にする
- 一覧画面では、画面全体ではなく記事一覧やタグなど必要な領域でスクロールする
- 記事一覧やカレンダー内のスクロールバーは控えめに表示し、スクロールバーの有無でカード幅が変わらないようにする
- 詳細画面は閲覧 / 編集の切り替えで本文領域幅が変わらないようにする
- 学習継続のモチベーションが上がるよう、過度に事務的でないUI表現を取り入れる
- 学習継続メッセージは固定表示にせず、複数のコメントと画像をアクションに合わせてローテーションする
- レスポンシブ確認では `docs/designs/responsive.md` の標準確認サイズと診断スクリーンショット手順を使う
