# 画面・運用の非機能要件

- デスクトップ、ノート PC、タブレット、スマホで主要導線を操作できるレスポンシブ UI を維持する
- データ永続化を前提とした信頼性を確保する
- 機能追加時に既存構成を保ちやすい設計にする
- 主要画面では操作中にレイアウトが不自然にずれず、長いコンテンツでも主要操作を継続できる状態を維持する
- 学習継続のモチベーションが上がるよう、過度に事務的でないUI表現を取り入れる

具体的なレイアウト、スクロール領域、学習継続メッセージの表示仕様は [UI仕様](../../specs/ui/README.md) と [デザイン文書](../../designs/README.md) を正本とします。
レスポンシブ確認手順は [レスポンシブ対応](../../designs/responsive/README.md) と [レスポンシブ診断キャプチャ](../../designs/responsive/capture.md) に従います。
