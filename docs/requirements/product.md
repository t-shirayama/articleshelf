# プロダクト要件

## ターゲットユーザー

- 技術記事を日常的に読むエンジニア
- 学んだ内容を蓄積しておきたい人
- 「あとで読む」すら消化できない人
- Qiita / Zenn / Medium などをよく利用する人

## 主な課題

- ブックマークが散らかりやすい
- 読んだ記事を忘れてしまう
- 「あとで読む」が消化されない
- メモと記事が分離している
- 既読履歴の振り返りが面倒

## 解決すること

- 記事をアプリ内で一元管理する
- URLからOGPを取得して記事情報を補完する
- 既読ステータスと既読日で整理する
- タグによるカテゴリ分けを可能にする
- 記事ごとにメモを残せるようにする
- 検索とフィルタで必要な記事をすばやく見つける
