# Improve auth form card contrast

## 状態

未対応

## 優先度

P3

## 目的

ログイン / ユーザー登録フォームの角丸・影付きカードが背景に少し溶け込んで見えるため、背景との差を少し強めてフォーム領域の視認性と操作対象としての分かりやすさを高める。

## 対象

- `frontend/src/features/auth/components/AuthScreen.vue`
- 認証画面まわりの CSS
- `docs/designs/components/auth.md`
- `docs/specs/ui/README.md`
- `docs/designs/screenshots/`
- 認証画面の desktop / tablet / mobile 表示

## 対応内容

- ログイン / ユーザー登録フォームカードの背景色、border、shadow、角丸、周囲余白を調整する。
- 背景から浮かせすぎず、ArticleShelf の落ち着いたトーンを維持したままコントラストを上げる。
- 入力欄、セグメントコントロール、主要ボタンの視認性と階層が崩れないように確認する。
- 左側のヒーロー / モックアップ UI とフォームカードの主従関係を保つ。
- 変更後の design docs、UI specs、スクリーンショット、テスト期待値の更新要否を確認する。

## 完了条件

- デスクトップ幅でログイン / 登録フォームカードが背景から自然に分離して見える。
- カードの角丸、影、border が過度に強くならず、既存デザインと調和している。
- タブレット / スマホ幅でフォームカードが見切れたり、背景と同化したりしない。
- 日本語 / English の長い文言や入力エラー表示でもカード内の余白が崩れない。
- 認証画面の design docs、UI specs、スクリーンショットが現行 UI と一致する。

## 根拠

認証画面ではフォームが主要操作領域になるため、背景とカードの境界が弱いとユーザーの視線が散りやすい。
少しだけコントラストを強めることで、ログイン / 登録の入力開始位置をより明確にできる。
