# 詳細編集でステータス変更時に既読日を同期する

## 状態

未対応

## 優先度

P2

## 目的

記事詳細ページの編集でステータスを変更したとき、既読状態と既読日の整合性を保つ。

## 対象

- 記事詳細ページの編集モード
- 記事更新 API に渡す `status` / `readDate`
- 記事詳細編集に関する feature / UI 仕様

## 対応内容

- 詳細ページの編集でステータスを未読から既読に変更したとき、既読日が未設定なら操作日を設定する
- 詳細ページの編集でステータスを既読から未読に戻したとき、既読日を未設定にする
- 既読へ変更する時点で既読日がすでに設定されている場合は、その日付を維持する
- 記事カード上の未読 / 既読切り替えと、詳細編集時の既読日同期ルールを揃える
- 実装時は `docs/specs/features/articles.md` と `docs/specs/ui/README.md` の詳細編集仕様を更新する
- 必要に応じて、詳細編集に関する frontend unit / E2E の期待値を更新する

## 完了条件

- 詳細編集でステータスを既読にしたとき、既読日未設定なら当日が設定される
- 詳細編集でステータスを未読に戻したとき、既読日が未設定になる
- 既読日が設定済みの状態で既読へ変更しても、既存の既読日が上書きされない
- 記事カード上のステータス切り替えと詳細編集時の挙動が矛盾しない
- 関連する feature / UI 仕様が現行挙動として更新されている

## 根拠

記事カード上では、未読から既読へ切り替えると既読日は操作日になり、既読から未読へ戻すと既読日は未設定になる。
詳細ページの編集でも同じルールに揃えることで、既読ステータスと既読日の不整合を防ぎ、ユーザーが手入力で補正する手間を減らせる。
