# PDF インポート実データ品質改善

## 状態

未対応

## 優先度

中

## 目的

PDF 取り込みで実データを扱うときの抽出品質、エラー扱い、確認しやすさを改善し、記事情報を手作業で補正しやすくする。

## 対象

- PDF からの記事情報抽出
- 抽出結果のタイトル、URL、概要、タグ、メモ候補
- 抽出失敗時や低信頼度時の UI / エラー表示
- 実データを使った品質確認観点

## 対応内容

- 実データで起きやすい抽出失敗パターンを整理する
- 抽出結果の信頼度や欠落項目をユーザーが判断できる表示を検討する
- URL、タイトル、メモ候補を手動補正しやすい入力フローを整理する
- 品質確認に使う代表ケースと期待結果を定義する

## 完了条件

- PDF インポートの実データ品質課題が分類されている
- 抽出結果の補正フローとエラー表示方針が仕様またはデザイン docs に反映されている
- 代表的な確認ケースが testing docs またはタスク完了要約に残っている
- 完了時に `archive/YYYY-MM.md` へ要約を追記し、このタスクファイルを削除している

## 根拠

PDF 由来の情報は記事 URL 直接登録よりも抽出の揺れが大きく、実データでの品質確認と補正導線を個別タスクとして扱う必要があるため。
