# Refine auth hero mockup layout

## 状態

未対応

## 優先度

P3

## 目的

ログイン / ユーザー登録画面の左側にあるモックアップ UI が一部重なって見え、情報が散らかった印象になっているため、配置、余白、重なり方を整理して視認性を上げる。

## 対象

- `frontend/src/features/auth/components/AuthScreen.vue`
- 認証画面まわりの CSS
- `docs/designs/components/auth.md`
- `docs/specs/ui/README.md`
- `docs/designs/screenshots/`
- 認証画面の visual / responsive 確認

## 対応内容

- 左側の価値訴求エリアにある進捗、最近保存した記事、人気タグなどのモックアップ UI の重なりと余白を見直す。
- デスクトップ幅で各モックアップ要素が読めるよう、z-index、位置、サイズ、影、背景透過を整理する。
- タブレット / スマホでは左側ビジュアルが表示縮小または非表示になる既存レスポンシブ挙動を崩さない。
- ログイン / ユーザー登録フォーム側の視線誘導を邪魔しないよう、左側ビジュアルの情報量を抑える。
- 変更後の認証画面スクリーンショットと design docs / UI specs の更新要否を確認する。

## 完了条件

- デスクトップ幅のログイン画面で左側モックアップ UI の重なりが不自然に見えない。
- 左側ビジュアルが整理され、ログイン / 登録フォームの操作を妨げない。
- 主要デスクトップ、タブレット、スマホ幅でレイアウト崩れがない。
- 認証画面の design docs、UI specs、スクリーンショットが現行 UI と一致する。

## 根拠

現状の左側モックアップ UI は保存、タグ、進捗の価値を伝える役割を持つ一方、要素同士が近く見えすぎると画面全体が散らかった印象になり、認証フォームへの集中を弱める。
