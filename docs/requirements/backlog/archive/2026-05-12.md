# 2026-05-12 Backlog Archive

2026年5月12日に完了した Backlog タスクの要約を記録します。

## 完了タスク

- Application logging design: 運用ログ、監査ログ、ユーザー向け履歴機能の責務境界を quality / security / backend architecture / testing docs に整理した。`X-Request-Id` で frontend の API failure と backend の request / exception を突き合わせる前提、auth / OGP / article update などの高価値イベント分類、構造化ログの基本項目、禁止情報と allowlist ベースの redaction 方針、環境別 log level の考え方を明文化した。
- JWT token service time id injection: `JwtTokenService` の access token 発行時刻と `jti` 生成を injected `Clock` / `IdGenerator` に統一し、`JwtTokenServiceTest` で `iat` / `exp` / `jti` の固定値検証を追加した。auth / security / backend architecture / testing docs も、access token 発行が refresh token 系と同じ deterministic provider 方針に従う形へ同期した。
- Auth use case bean injection: `AuthService` から `new RefreshTokenRotationService(...)` と `new InitialUserProvisioner(...)` を外し、Spring bean injection された helper を受け取る orchestration service へ整理した。`AuthServiceTest` も helper を明示的に渡す形へ更新し、auth specs、backend architecture、testing docs に DI 境界を反映した。
- Detail page shared sidebar navigation: `ArticleWorkspaceShell` を detail route でも維持するようにして、desktop では sidebar を残したまま詳細を表示し、mobile では header / drawer を継続利用しつつ bottom navigation を detail 中に隠す形へ整理した。detail が一覧起点なら一覧フィルタの active state、calendar 起点なら calendar active と戻り先を保つようにし、frontend architecture / design / UI / testing docs を同期した。
- Frontend focused coverage gates: `vitest.focused-coverage.config.ts` と `npm run test:unit:coverage:focused` を追加し、articleFilters、calendar、renderMarkdown、API client、JWT decode、proactive refresh timer、主要 article composable を global coverage とは別の高い threshold で確認できるようにした。testing / CI docs も global coverage と focused coverage の役割分担へ同期した。
